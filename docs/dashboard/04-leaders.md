## 功能说明

跟单的核心是 **Leader**（被跟单者）。Dashboard 提供两种添加方式：

| 方式 | 适合场景 |
|------|----------|
| **发现** 页浏览排行榜 | 不知道跟谁，想从盈利榜挑选 |
| **Leaders** 页手动添加 | 已有目标地址 / 用户名 |

---

## 发现 Trader

**路径：** 发现

### 筛选维度

| 维度 | 选项 |
|------|------|
| 分类 | 综合、政治、体育、加密、金融、科技、文化 |
| 周期 | 24h / 7d / 30d / 全部 |
| 排序 | PnL、成交量等 |

### 操作流程

1. 浏览排行榜，点击 Trader 进入详情
2. 查看历史 PnL、胜率、近期活动
3. 点击 **添加为 Leader**，填写 ID 与策略
4. 保存后跳转 **Leaders** 页，可继续微调

> 数据来自 Polymarket 官方 API，缓存约 60 秒。加载失败请先配置代理。

## 管理 Leader 列表

**路径：** Leaders

### 列表字段

| 列 | 说明 |
|----|------|
| ID | 配置中的唯一标识 |
| 地址 / 用户名 | `0x…` 或 `@handle` |
| 启用 | 点击 ✓ / — 快速启停 |
| 跟单设置 | 行内编辑比例与单笔上限 |
| 策略类型 | PERCENTAGE / FIXED / ADAPTIVE |
| 权重 | 冲突处理时的优先级 |
| 今日成交额 | 当前账户下该 Leader 今日跟单量 |

### 行内快捷编辑

在 **跟单设置** 列直接修改：

| 字段 | PERCENTAGE 模式 | FIXED 模式 |
|------|-----------------|------------|
| 比例 / copy | Leader 成交额 × N% | 每笔固定 $N |
| 上限 $ | 单笔跟单 USD 上限 | 同上 |

修改后 **失焦或按 Enter** 自动保存，写入 `config.yaml` 并热重载。

## 新建 / 编辑 Leader

**路径：** Leaders → **+ 添加 Leader** 或行内 **编辑**

### 基本信息

| 字段 | 说明 |
|------|------|
| ID | 唯一标识，建议简短英文 |
| 模式 | 地址模式 或 用户名模式 |
| address | `0x` 开头的钱包地址 |
| username | Polymarket `@handle`（不含 @） |

### 策略配置

| 类型 | copy_size 含义 | 典型场景 |
|------|----------------|----------|
| `PERCENTAGE` | Leader 成交额 × N% | 大多数用户首选 |
| `FIXED` | 每笔固定 $N | 控制绝对仓位 |
| `ADAPTIVE` | 大单缩比、小单放大 | 跟随大户但限制极端单 |

### 过滤与限制（高级）

完整表单中可配置：

| 类别 | 示例 |
|------|------|
| 价格过滤 | `min_price` / `max_price` |
| 方向 | 仅 BUY / 仅 SELL |
| 单笔上限 | `max_order_usd` |
| 权重 | 多 Leader 冲突时的优先顺序 |

## 跟单逻辑简述

引擎每个轮询周期：

```
拉取 Leader 近期成交 → 去重 → 过滤 → 缩放仓位 → 风控检查 → 下单（或 Preview 模拟）
```

未跟单时在 **活动流** 查看 `SKIP` 及原因（价格过滤、超限额、Kill Switch 等）。

---

## 常见操作

| 目标 | 操作 |
|------|------|
| 暂停跟某人 | Leaders 列表点击启用列切换为 — |
| 减小跟单力度 | 降低比例 % 或单笔上限 $ |
| 换 Leader | 禁用旧的，添加新的 |
| 排查无跟单 | 活动流筛选该 Leader + 动作 SKIP |
