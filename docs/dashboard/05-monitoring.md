## 功能说明

运行监控模块帮助你在跟单过程中 **掌握引擎状态、持仓与决策细节**。建议日常巡检顺序：

```
总览（健康） → 活动流（有无跟单） → 持仓 / 订单（结果）
```

---

## 总览

**路径：** 总览（侧边栏首页）

### 状态指标

| 指标 | 说明 |
|------|------|
| Preview / Live | 当前运行模式 |
| Kill Switch | 风控熔断是否激活 |
| 运行时长 | 引擎 uptime |
| 今日跟单 / 成交额 | 当日 COPY 统计 |
| 最近错误 | 轮询或下单异常 |

### 多账户视图

配置多个账户时，总览顶部展示 **全部账户卡片**，便于对比各账户今日成交额与 Leader 数量。

### 最近审计

页面底部列出最近审计事件，可快速跳转 **活动流** 查看详情。

---

## 持仓

**路径：** 持仓

展示 **引擎本地 SQLite** 跟踪的跟单持仓，按 Leader 分组。

| 列 | 说明 |
|----|------|
| Token | 市场 token ID |
| 份额 | 当前持有量 |
| 均价 | 加权平均入场价 |
| 估算成本 | 份额 × 均价 |

### 与「我的账户」的区别

| 页面 | 数据来源 | 说明 |
|------|----------|------|
| 持仓（本页） | 引擎 SQLite | 跟单引擎视角 |
| 我的账户 → 引擎跟踪 | 同上 | 同一数据源 |
| 我的账户 → Polymarket 持仓 | 链上 API | 官网视角 |

Preview 模式下本页会有模拟持仓，Polymarket 链上可能仍为 0。

---

## 订单

**路径：** 订单

展示尚未完全成交的 **GTC 限价单**（Pending）。

| 列 | 说明 |
|----|------|
| 已成交 / 总量 | 部分成交进度 |
| 挂单时长 | 从创建至今 |
| Order ID | CLOB 订单号 |
| 操作 | Live 模式下可 **撤单**（调用 CLOB cancel） |

无 Pending 时表格为空，属正常。Live 模式下限价单未立即吃单时会出现在此。

---

## 活动流

**路径：** 活动流

跟单决策的 **完整审计日志**，排查问题的首选页面。页面通过 **SSE 实时推送** 新事件，无需手动刷新；同时每 15 秒轮询兜底。

### 筛选

| 筛选项 | 用途 |
|--------|------|
| Leader | 只看某个 Leader |
| 动作 | DETECT / SKIP / COPY / ERROR |

### 动作含义

| 动作 | 含义 |
|------|------|
| `DETECT` | 检测到 Leader 新成交 |
| `SKIP` | 决定不跟（看原因列） |
| `COPY` | 已跟单（Preview 或 Live） |
| `ERROR` | 处理过程出错 |

### 常见 SKIP 原因

| 原因类型 | 处理建议 |
|----------|----------|
| 价格过滤 | 调整 Leader 的 min/max price |
| 低于 min_order | 提高 copy_size 或策略比例 |
| 超过 max_order | 正常截断，或调低 Leader 上限 |
| 超日限额 | 设置 → 风控，或等次日 |
| Kill Switch | 风控页重置 |
| 重复成交 | 去重窗口内已处理，可忽略 |

标记 `[P]` 表示 Preview 模拟记录。

---

## 监控建议

| 频率 | 检查项 |
|------|--------|
| 每日 | 总览 Kill Switch、今日成交额 |
| 添加 Leader 后 | 活动流是否有 DETECT |
| 开 Live 前 | 连续数日 COPY 符合预期 |
| 异常时 | 活动流 ERROR + 总览最近错误 |
