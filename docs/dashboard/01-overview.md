## 产品概览

PolyMirror 是运行在本地或 VPS 上的 **Polymarket 跟单引擎**。Dashboard 是它的 **Web 管理控制台**，用于配置 Leader、查看持仓与 PnL、切换 Preview/Live，无需手改 `config.yaml`。

### 核心能力

| 能力 | 说明 |
|------|------|
| 多 Leader 跟单 | 同时跟踪多个地址 / 用户名 |
| 多账户 | 单进程管理多个钱包，数据与 Leader 隔离 |
| Preview 模拟 | 默认不真实下单，先观察再实盘 |
| 策略缩放 | 按比例 / 固定金额 / 自适应 |
| 风控 | 日限额、Kill Switch、单笔上下限 |

### Dashboard 能做什么 / 不能做什么

| 能做 | 不能做 |
|------|--------|
| 添加 / 编辑 Leader 与策略 | 代替 Polymarket 官网手动下单 |
| 管理钱包与多账户 | 托管或上传私钥到云端 |
| 查看 PnL、持仓、审计日志 | 跨平台（Kalshi 等）跟单 |

## 推荐使用流程

按以下顺序操作，符合大多数用户从试跑到实盘的习惯：

```
① 安装启动 → ② 配置钱包 → ③ 添加 Leader → ④ Preview 观察
       ↓
⑤ 调整策略与风控 → ⑥ 确认活动流正常 → ⑦ 切换 Live（小额）
```

| 阶段 | 对应页面 | 目标 |
|------|----------|------|
| ① 安装启动 | — | 引擎与 Dashboard 正常运行 |
| ② 配置钱包 | **我的账户** | 私钥、Proxy 地址正确 |
| ③ 添加 Leader | **发现 Trader** / **Leader 管理** | 至少 1 个启用的 Leader |
| ④ Preview 观察 | **总览** / **活动流** | 确认有 DETECT / COPY 记录 |
| ⑤ 调整策略 | **Leader 管理** / **设置** | 比例、限额、轮询间隔 |
| ⑥ 排查异常 | **活动流** | 查看 SKIP 原因 |
| ⑦ 实盘 | **设置 → 模式** | 二次确认后开 Live |

> **建议：** Preview 至少运行数天，确认跟单逻辑符合预期后再开 Live，并使用 **专用小额钱包**。

## 界面结构

侧边栏按 **工作台 / 交易 / 系统** 三组排列，与控制台导航一致：

| 分组 | 页面 | 用途 |
|------|------|------|
| **工作台** | **总览** | 运行状态、今日统计、多账户摘要 |
| | **活动流** | 跟单决策审计（DETECT / SKIP / COPY） |
| **交易** | **发现 Trader** | Polymarket 排行榜、一键加 Leader |
| | **Leader 管理** | Leader 列表、策略、启停 |
| | **持仓** | 引擎 SQLite 跟踪的跟单持仓 |
| | **订单** | Pending GTC 限价单、撤单 |
| **系统** | **我的账户** | 钱包资料、PnL、多账户管理 |
| | **风控** | Kill Switch、限额进度 |
| | **设置** | 全局参数、代理、Preview/Live |
| | **操作文档** | 本帮助中心 |

### 账户切换器

侧边栏顶部的 **账户下拉框** 决定当前操作上下文：

- 所有页面数据（Leader、持仓、统计）按所选账户隔离
- 切换账户后页面会刷新
- 每个账户有独立的 SQLite 与 `.env` 钱包变量
